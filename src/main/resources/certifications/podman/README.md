## 📋 Description

Podman est un moteur de conteneurs sans démon, conçu pour les environnements Linux et compatible avec les images Docker. La certification **Podman Professional Red Hat** atteste de votre maîtrise de la gestion de conteneurs en mode rootless, de la manipulation des pods (groupes de conteneurs), de la persistance des données avec les volumes, et de l'intégration native avec systemd et Kubernetes.

Elle est indispensable pour les administrateurs système et les ingénieurs DevOps souhaitant adopter une approche plus sécurisée et légère que Docker.

## 🏛️ Organisme délivrant

**Red Hat, Inc.** – L’examen pratique est administré par Red Hat.

## 📅 Validité

La certification est **valide 3 ans**. Renouvellement possible par examen ou obtention d'une certification supérieure.

## 📚 Programme détaillé

| Thème | Compétences évaluées |
|-------|----------------------|
| **Fondamentaux de Podman** | Architecture sans démon, mode rootless vs rootfull, commandes de base. |
| **Gestion des conteneurs et pods** | Création, exécution, inspection, logs, exécution de commandes dans un conteneur, gestion des pods. |
| **Gestion des images** | Recherche, pull, build (Containerfile/Dockerfile), tag, push vers des registres. |
| **Réseau et volumes** | Création de réseaux, exposition de ports, gestion des volumes persistants et bind mounts. |
| **Intégration systemd et Kubernetes** | Génération de fichiers unit systemd pour conteneurs/pods, déploiement de pods sur Kubernetes via `podman generate kube` et `podman play kube`. |
