## Identité du projet

Tu travailles sur CertifApp, la migration commerciale de CertiPrep Engine.
CertiPrep Engine est une application JavaFX de bureau de préparation aux certifications professionnelles.

Fonctionnalités existantes (à migrer intégralement) :
- 4000+ questions couvrant une dizaine de certifications complètes
- 500+ questions spécifiques Java 21 Developer (OCP)
- Mode Examen : conditions réelles (temps limité, notation officielle)
- Mode Libre : personnalisation des thèmes, nombre de questions, durée
- Mode Révision : apprentissage avec correction immédiate
- Thèmes clair/sombre commutables
- Internationalisation Français/Anglais déjà implémentée
- Historique des sessions avec SQLite (à migrer vers PostgreSQL)
- Export PDF des résultats
- Import/Export de questions supplémentaires (format à identifier dans le code)

Stack actuelle à migrer : JavaFX + Java 17/21 + SQLite

Objectif : migration complète vers Spring Boot 3 (backend REST), Angular 18 PWA (web)
et Android Kotlin/Compose (mobile), destinée à une commercialisation sur l'ensemble
du marché francophone mondial.

## Phase Enrichissement IA des questions (phase dédiée)

Cette phase intervient après la migration technique. Elle constitue un cycle
de relecture et d'enrichissement automatisé du corpus de 4000+ questions.

### Objectif
Améliorer la qualité pédagogique de chaque question en enrichissant la section
"Explication de réponse" avec :
- Justification détaillée de la bonne réponse (pourquoi c'est correct)
- Explication de pourquoi chaque mauvaise option est incorrecte (distracteurs)
- Référence à la documentation officielle (lien Oracle, AWS docs, etc.)
- Exemple de code illustratif si pertinent (questions Java/Spring)
- Lien vers le chapitre officiel de la certification concernée
- Niveau de difficulté revu si nécessaire

### Processus technique à implémenter
- Interface d'administration dédiée : liste des questions avec statut de relecture
  (NON_RELU / EN_COURS / RELU_VALIDE / RELU_ENRICHI)
- Enrichissement IA assisté : bouton "Enrichir via IA" par question ou par lot
  → appel LangChain4j → Claude/Ollama génère l'enrichissement
  → affichage côte à côte : version originale vs version enrichie
  → validation humaine obligatoire avant sauvegarde (human-in-the-loop)
- Enrichissement par lot : sélection d'un domaine ou d'une certification entière
- Historique des modifications : chaque version de l'explication est versionnée
- Statut de confiance IA : score de certitude affiché pour guider la relecture humaine

### Modèle de données enrichi
Chaque explication doit supporter :
- explanation_original   : texte original importé depuis JavaFX
- explanation_enriched   : version enrichie par IA ou manuellement
- explanation_status     : enum (ORIGINAL / AI_DRAFT / HUMAN_VALIDATED)
- explanation_version    : numéro de version
- last_reviewed_by       : userId du relecteur
- last_reviewed_at       : timestamp
- ai_confidence_score    : Float (0.0 à 1.0)
- official_doc_url       : lien documentation officielle
- code_example           : bloc de code illustratif (optionnel)

## Fonctionnalités innovantes à implémenter

### IA pédagogique — différenciation marché

1. Parcours adaptatif intelligent
   Algorithme de répétition espacée (style Anki/SM-2) adapté aux certifications :
   - Analyse des erreurs passées par domaine
   - Priorisation automatique des domaines faibles
   - Prédiction du score à l'examen en temps réel
   - Jour J recommandé pour passer la certification

2. Explication interactive par IA
   Bouton "Expliquer cette question" sur chaque réponse incorrecte :
   - LangChain4j appelle Claude/Ollama avec le contexte de la question
   - Réponse en langage naturel, personnalisée selon l'historique de l'utilisateur
   - Suggestions de questions similaires pour consolider la notion

3. Génération de questions à la demande (admin + premium users)
   - L'admin peut générer de nouvelles questions sur un domaine donné via IA
   - Validation humaine obligatoire avant publication (human-in-the-loop)
   - Les utilisateurs premium peuvent demander des questions sur un concept précis

4. Assistant de révision conversationnel
   Chat intégré (interface style messagerie) propulsé par LangChain4j :
   - Répond aux questions sur les notions de la certification
   - Génère des quiz flash à la demande ("donne-moi 5 questions sur les streams Java")
   - Contexte maintenu sur la session de révision
   - Disponible offline sur mobile (modèle quantisé embarqué ou cache de réponses)

5. Analyse prédictive et tableau de bord intelligent
   - Score de préparation par domaine (jauge 0-100%)
   - Courbe de progression hebdomadaire
   - Comparaison anonymisée avec d'autres candidats (benchmarking)
   - Alertes : "Tu n'as pas révisé Lambda Expressions depuis 7 jours"
   - Rapport PDF intelligent généré par IA avant l'examen

6. Mode duel / compétition (gamification)
   - Défi entre deux utilisateurs sur un domaine donné
   - Classements par certification et par zone géographique
   - Badges et certifications de progression
   - Partage des scores sur LinkedIn (intégration OAuth LinkedIn)

7. Import intelligent de questions
   - Upload d'un PDF (dump, livre de préparation) → extraction automatique des questions par IA
   - Validation et correction par l'admin avant intégration au catalogue
   - Détection des doublons par similarité sémantique (embeddings)

8. Mode simulation d'examen officiel
   - Reconstitution exacte des conditions Pearson VUE / Prometric
   - Interface identique à l'examen officiel (layout, navigation, timer)
   - Rapport post-examen avec analyse par domaine conforme au score report officiel

### Fonctionnalités de distribution et croissance

9. Programme de contribution communautaire
   - Les utilisateurs peuvent proposer de nouvelles questions
   - Système de vote et validation communautaire
   - Crédit/récompense pour les contributeurs (accès premium offert)

10. API publique pour partenaires
    - Accès au catalogue de questions via API pour écoles, bootcamps, entreprises
    - Facturation à l'usage (Stripe Metered Billing)
    - Tableau de bord partenaire avec statistiques d'utilisation
